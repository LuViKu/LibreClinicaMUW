Designing the next LibreClinica
for the Department of Ophthalmology & Optometry.

Modern, clinical, data-dense — lineage drawn from REDCap, Castor EDC and Medidata Rave, dressed in the Medical University of Vienna corporate design. Seventeen mockups, each replacing a specific legacy JSP in the current LibreClinica build.

MUW v1
Investigator muw-teal

Physician / study assistant at a site. Adds subjects, schedules events, enters CRF data, handles discrepancies, signs subjects.

Monitor muw-sky

Source Data Verification, query authority — the only role that can close discrepancies. Read-only access to data.

Data Manager muw-coral

Study setup: Build Study workflow, CRF design, event definitions, rules, sites, users. Broadest surface area.

Round 1 · Critical workflows

One per role

7 screens
Investigator /ListStudySubjects
Subject Matrix
Site-scoped subject list with per-event status. Search, filter, schedule new event, sign subject.
Investigator /AddNewSubject
Add Subject
Enrol a study subject. Three save modes: finish, add next, schedule first event.
Investigator /InitialDataEntry
CRF data entry
Per-section CRF form with inline discrepancy threads, required-field markers, mark-complete.
Monitor /pages/viewAllSubjectSDVtmp
SDV table — View by Event & CRF
Completed CRFs across all subjects with SDV requirement, status, bulk-verify action.
Monitor /ViewSectionDataEntry
Read-only CRF view
The popup primitive opened from SDV. Item-level flags, side-panel discrepancy thread.
Data Manager /pages/studymodule
Build Study
The 7-task setup tracker — Create Study → CRF → Events → Groups → Rules → Sites → Users.
Data Manager /UpdateEventDefinition
Update Event Definition
Event metadata + per-CRF assignment with SDV requirement (the value Monitor sees).

Round 3 · Sign-off, modal, audit, designer

High-stakes surfaces

6 screens
Investigator /SignStudySubject
Sign Subject (M-001)
Pre-flight checklist, casebook snapshot, e-signature attestation. Pattern: one-way regulatory action.
Monitor /CreateDiscrepancyNote
Add Query modal
Modal over dimmed CRF context. Segmented note-type, severity, assign, due date. Pattern: focused-action modal.
Inv Mon DM
/ViewAuditLog
Study Audit Log
Timeline of every regulated event with before/after diff cards. Filters narrow, never hide.
No auth /pages/login/login
Login + forced password change
Centred card with MUW lockup, LDAP fallback. Right pane: first-login stepper (password, 2FA, terms).
Data Manager /CreateCRF · /EditCRF
Create / Edit CRF
Item-grid designer, section rail, response-type cells, repeating groups, response-set side panel, live Investigator preview.
Data Manager /ImportCRFData
Import CRF Data wizard
Multi-step: Upload → Map → Preview & resolve → Commit. Per-row before/after diff with action selector.
Heads up

These are static sketches.

Filters, buttons and form fields look real but do nothing. Open design-notes.md for the design tokens, the scope decisions, and what is explicitly NOT in the first pass (empty / error / loading states, real interactivity, responsive breakpoints, accessibility audit, i18n).
